2. Create `src/components/ComparisonTable.astro`:
   - Props: `entries` (array of critter data objects from getCollection)
   - DESKTOP view (hidden on mobile via `hidden md:block`):
     - Table-like grid layout using `grid grid-cols-6` (Name, Diet, Output, Rate, Stable Cap, DLC)
     - Header row: bold, `text-oni-text-muted text-sm uppercase tracking-wider`
     - Data rows: alternating subtle backgrounds (`even:bg-oni-bg-surface/50`), border-b
     - Name column: `font-medium text-oni-text-primary`
     - Diet column: comma-joined list, `text-oni-text-secondary`, truncate if long
     - Output column: product name
     - Rate column: use StatPill component for the numeric value
     - Stable capacity: use StatPill
     - DLC column: small badge/tag showing DLC status (e.g., "Base" in a subtle rounded badge)
   - MOBILE view (shown only on mobile via `md:hidden`):
     - Each entry as a stacked card: `bg-oni-bg-surface border border-oni-bg-elevated rounded-lg p-4 mb-3`
     - Name as card title, followed by definition list (dl) with key stats
     - Use StatPill for numeric values
     - DLC badge in top-right corner of card

3. Create `src/components/DetailCard.astro`:
   - Props: `entry` (single critter data object)
   - Uses HTML `<details>` element for native expand/collapse (no JS needed, accessible)
   - `<summary>`: critter name + brief identifier (e.g., "Hatch -- Coal Producer"), styled as clickable header with `cursor-pointer`, hover highlight, chevron indicator
   - Expanded content: full stat breakdown in a grid layout
     - Diet: full list of food items, each as a styled tag/chip
     - Output: product name + rate (StatPill)
     - Metabolism: kcal/cycle (StatPill)
     - Temperature range: min to max (StatPill for each)
     - Stable capacity: number (StatPill)
     - Grooming: station name + cycle time
     - DLC compatibility: badge(s)
     - Notes: if present, displayed in a subtle callout box with different background
   - Card border: `border border-oni-bg-elevated rounded-xl overflow-hidden`
   - When open: `open:border-oni-teal-dim` for visual feedback
   - Placeholder image slot: a colored rectangle (CSS only, no image file) with critter name as alt text, positioned top-right or as a subtle background element. Use `bg-oni-bg-elevated` with a centered emoji or icon placeholder.
